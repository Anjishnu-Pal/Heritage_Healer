# Outputs Directory

This directory holds all artifacts produced by training and inference across the three model approaches. The subfolders mirror the `src/` layout.

---

## Diffusion Model

**Folder:** `outputs/Diffusion model/`

| Artifact | Generated by | Description |
|---|---|---|
| `diffusion_photo_restorer_epoch_N.pth` | `train.py` | UNet2D state dict saved after each epoch |
| `diffusion_photo_restorer/logs/` | `train.py` | TensorBoard training logs |
| `restored_output.jpg` | `inference.py` | Single restored image from inference |

Weights are saved after every epoch — set `LAST_EPOCH` in `inference.py` to pick which checkpoint to load.

---

## GANs (pix2pix)

**Folder:** `outputs/GANs/`

| Artifact | Generated by | Description |
|---|---|---|
| `training_checkpoints/ckpt-N.index` | `train.py` | TensorFlow checkpoint index file |
| `training_checkpoints/ckpt-N.data-*` | `train.py` | TensorFlow checkpoint data file |
| `training_checkpoints/checkpoint` | `train.py` | Checkpoint metadata (tracks latest) |
| `restored_output.jpg` | `inference.py` | Single restored image from inference |

Checkpoints are saved every 5 epochs. `inference.py` loads the latest checkpoint automatically via `tf.train.latest_checkpoint()`.

---

## VAEs

**Folder:** `outputs/VAEs/`

| Artifact | Generated by | Description |
|---|---|---|
| `model6.h5` | `train.py` | Full Keras model (architecture + weights) |
| `restored_output.jpg` | `inference.py` | Single restored image from inference |

The `.h5` file contains the full model and can be reloaded with:
```python
keras.models.load_model('model6.h5', custom_objects={
    'Reparameterization': Reparameterization,
    'vae_loss': vae_loss
})
```

---

## Notes

- Model weight files (`*.pth`, `*.h5`, `*.keras`) and TF checkpoint folders (`training_checkpoints/`) are excluded from version control via `.gitignore` due to their size.
- Validation plots (`.png`, `.jpg`) generated during training are displayed inline and not persisted to disk.
- The `.gitkeep` files in each subfolder exist only to preserve the folder structure in git.
